Translations

Hawaii is created for all people around the World;
help us make it available to everyone into their language.

Do you want to help translate Hawaii?

This is awesome! We're always in need of Translators.

If a translation team for you language already exist, join their effort regardless. There will be more strings to translate in the future, also reviewing existing translations would be very helpful.

All Hawaii Translators must create an account on Transifex as their service powers our translation management.

How to become a Hawaii Translator

English is the source language. You must be able to translate from English to your desired language.

The steps are:

  1. Signup for an account on Transifex.
  2. Verify your email address, log in and then go to the Hawaii project page.
  3. Choose the language you want to translate or, if it isn't listed, click the Request Language button.
  4. If you are requesting a new language, select or search for it in the dialog box that will appear.
  5. If your desired language already exist, click on the Language link and then click the Join Team button on the next page.
  6. Click on the resource you want to work in.
  7. Click the Translate Now button to use the Transifex editor or, if you prefer, download the file for use offline and then upload your changes.
  8. If you are using the Transifex editor, choose the option "Untranslated" and you will see all the untranslated strings on the left bar.

Web

This Web site can be improved by anyone, we are waiting for your improvements and design.

Do you want to contribute to the Hawaii Web site?

Brilliant! There is never enough contributors to the Web site.

This is the easiest module to contribute to. It is statically generated using Jekyll.

Pages are written either with Markdown or HTML, while styles are generated from Sass (people familiar with CSS won't have any problem with it).

How to become a Hawaii Web developer

You must be familiar with GitHub pull requests and with the technologies mentioned above.

The steps are:

  1. Signup for an account on GitHub.
  2. Verify your email address, log in and then go to the Web site repository.
  3. Fork the repository by clicking on the Fork button, once done it will automatically open the page with the fork.
  4. Open a terminal
  5. Clone your fork (the address is in the fork repository page).
  6. Install the development tools as explained in this paragraph.
  7. Serve the pages locally with: 
    jekyll serve
  8. Edit the pages and test them in the browser pointing to localhost:4000.
  9. Once you are done commit the changes and push to your repository, then make a pull request.

Test

Making high quality software requires lots of testing.
Reporting errors or possible improvements is a valuable way to help with the project and is a great way to start out.

Do you want to test Hawaii?

Wonderful! The more testers the better.

Go grab a nightly image, make a USB flash drive, boot your laptop or desktop PC with it and report issues here.

How to become a Hawaii Reporter

After having downloaded and tested Hawaii, you should report any issue that you might have encountered.

The steps are:

  1. Signup for an account on GitHub.
  2. Verify your email address, log in and then go to the issue tracker.
  3. Press the New Issue button and fill with details.
